-- Давайте сделаем консольную игру, интересную викторину! --

Базовый уровень

  1. Создайте конструктор, назовите его Question, он должен создавать объект с вопросом. Вопрос должен включать в себя:
    1. сам вопрос
    2. ответы, среди которых пользователь должен выбрать верный ответ (выберите подходящую структуру данных: массив, объект, ...)
    3. Правильный вариант ответа (Рекомендация: используйте номер ответа для определения верного варианта)
  2. Создайте 3-4 вопроса, используя конструктор
  3. Поместите все вопросы в массив
  4. Выберите один случайный вопрос и выведите его в консоль, вместе с вариантами ответов.

    Подсказка 1.
    Каждый вопрос должен иметь номер. Случайный вопрос - это вопрос под случайным номером.

    Подсказка 2.
    Каждый вариант ответа должен иметь номер.

    Подсказка 3.
    Создайте метод для всех объектов класса Question для выполнения этого задания.

  5. Используйте функцию prompt для запроса ответа от пользователя. Пользователь должен ввести номер корректного ответа. Номера для каждого из вариантов ответов должны выводится вместе с вариантами, то что реализовано в пункте 4.
  6. Проверьте, является ли введенный ответ корректным, и распечатайте в консоль сообщение о правильном или неправильном ответе.

    Подсказка.
    Создайте для этого дополнительный метод для объектов класса Question.

  7. Предусмотрите вариант использования кода этой игры в существующем сайте. Позаботьтесь о том чтобы переменные, функции и классы не пересекались с другим возможным кодом.

    Подсказка.
    Используйте специальную конструкцию которую мы проходили чтобы сделать код игры приватным и недоступным извне.

Экспертный уровень

  1. После того как выводится результат ответа (верно/неверно), отобразите следующий случайный вопрос. Таким образом игра никогда не завершится и будет продолжаться бесконечно.

    Подсказка.
    Для этого напишите функцию с показом вопроса и вызывайте ее после отображения результата.

  2. Будьте осторожны. После выполнения 8-го пункта, игра будет продолжаться бесконечно. Предусмотрите вариант остановки игры. Чтобы пользователь мог ввести в консоль "exit" или нажать на кнопку "Отмена" и выйти из игры.

    Подсказка.
    В этом случае проверяйте ответ пользователя, и если это не "exit" и не закрытие окна с вопросом через кнопку "Отмена", тогда не запускайте функцию для нового вопроса, из пункта 8.

  3. Отслеживайте прогресс, баллы пользователя чтобы сделать игру более интересной. Каждый раз когда пользователь отвечает верно, добавляйте по 1 баллу к его счету.
  4. Отобразите счет пользователя в консоли, сразу после того как он даст ответ. Создайте для этого новый метод.
